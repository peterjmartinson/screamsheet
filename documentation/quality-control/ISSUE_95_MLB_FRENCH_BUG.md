## Bug Report

### Root Cause

In `FrenchMLBContentProvider.generate()` (`src/screamsheet/providers/french_mlb_content_provider.py`), all three LLM lanes (A2 rewrite, B2/C1 rewrite, and lexicon) are submitted to a `ThreadPoolExecutor` **concurrently**. This means Lane C (the lexicon) receives the raw scraped source article text as input — not the French stories generated by Lanes A and B.

As a result, the vocabulary and idiomatic phrases on the back page are drawn from the original source material rather than the actual French prose the reader sees on the front page, making the lexicon feel disconnected and generic.

---

### Expected Behavior

The lexicon should be derived from the LLM-generated French text (Lane A + Lane B output), so that every word and phrase in **Le Lexique Essentiel** and **Les Tournures de Phrase** appears in one of the front-page stories.

---

### Fix

Change the execution order in `FrenchMLBContentProvider.generate()`:

1. Run Lanes A and B first (can remain concurrent with each other).
2. After both complete, pass their output text to Lane C.
3. Weight sourcing roughly **~1/3 from the A2 story** and **~2/3 from the B2/C1 story**, since the advanced article contains richer vocabulary and more idiomatic content.
4. Update `french_mlb_lexicon.txt` prompt to explicitly instruct the LLM to extract only from the supplied generated text, not from general baseball knowledge.

**Files to modify:**
- `src/screamsheet/providers/french_mlb_content_provider.py` — serialize execution so Lane C runs after Lanes A and B, passing their output as input
- `src/screamsheet/llm/prompts/french_mlb_lexicon.txt` — update prompt to restrict extraction to the provided generated text only
